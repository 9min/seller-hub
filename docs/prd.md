# PRD - Seller-Hub (커머스 파트너 통합 관리 대시보드)

## 1. 프로젝트 개요

- **목적:** 대형 커머스 플랫폼에 입점한 브랜드 파트너(셀러)가 상품, 주문, 매출 데이터를 한눈에 파악하고 효율적으로 비즈니스를 운영할 수 있도록 지원하는 B2B 어드민 웹 애플리케이션.
- **해결하는 문제:** 다수의 플랫폼에 분산된 매출/주문 데이터를 통합 관리하지 못해 의사결정이 지연되는 문제를 해결한다.
- **기술적 목표:**
  - 수만 건의 대규모 데이터를 브라우저 환경에서 지연 없이 처리하는 렌더링 성능 최적화.
  - 복잡한 상태(다중 필터, 정렬, 페이지네이션)를 안정적으로 관리하는 프론트엔드 아키텍처 설계.
  - 기획부터 디자인, 인프라 연동, 배포에 이르는 E2E 사이클 주도.

## 2. 목표 사용자

- **주요 사용자:** 커머스 입점 브랜드 관리자 및 MD.
- **사용자 특성:** 하루 수천 건 이상의 주문을 처리하고, 실시간 정산 데이터 및 광고 효율 지표를 확인하여 비즈니스 의사결정을 내려야 하는 실무자.

## 3. 핵심 기능

- [ ] **비즈니스 인사이트 대시보드 (Overview Dashboard)**
  - 주요 지표 위젯: 일간/주간 총 매출, 신규 주문 건수, 배송 지연 건수, 반품/교환율 실시간 표출.
  - 매출 추이 라인 차트: 기간별(일/주/월) 매출액 추이 변화 시각화.
  - 카테고리별 판매 비중 파이 차트: 인기 상품 및 카테고리 분포도 제공.
  - 차트 렌더링 시 애니메이션 최적화, 반응형 레이아웃 적용.

- [ ] **고성능 대용량 데이터 그리드 (Complex Data Grid)**
  - 주문/상품 리스트: 최소 5만 건 이상의 데이터를 테이블 형태로 렌더링.
  - 가상화(Virtualization): 화면에 보이는 영역의 로우만 렌더링하여 DOM 노드 갯수 제한.
  - 복합 필터링: 기간 설정, 주문 상태 다중 선택, 텍스트 검색.
  - 각 컬럼별 오름차순/내림차순 다중 정렬 기능.
  - URL 쿼리 파라미터와 전역 상태 동기화를 통한 필터 상태 유지.

## 4. 사용자 스토리

- 셀러로서 로그인 직후 오늘의 매출/주문/배송 현황을 한눈에 확인하고 싶다. 그래서 빠른 의사결정을 내릴 수 있다.
- 셀러로서 기간별 매출 추이와 카테고리별 판매 비중을 차트로 보고 싶다. 그래서 비즈니스 트렌드를 파악할 수 있다.
- 셀러로서 5만 건 이상의 주문 데이터를 지연 없이 스크롤하며 탐색하고 싶다. 그래서 특정 주문을 빠르게 찾을 수 있다.
- 셀러로서 다중 필터(기간, 주문 상태, 검색어)를 조합하여 원하는 데이터만 추출하고 싶다. 그래서 업무 효율을 높일 수 있다.
- 셀러로서 필터 상태가 URL에 반영되어 동료에게 공유할 수 있기를 원한다. 그래서 동일한 뷰를 함께 확인할 수 있다.

## 5. 기술 스택

- **프론트엔드:** Vite + React + TypeScript
- **백엔드(BaaS):** Supabase (PostgreSQL, Auth, Storage, Edge Functions)
- **상태 관리:** Zustand (클라이언트 전역 상태)
- **데이터 페칭:** TanStack Query (서버 상태 관리)
- **데이터 그리드:** TanStack Table + TanStack Virtual (가상화 렌더링)
- **차트:** Recharts (매출 추이, 카테고리 분석)
- **CSS:** Tailwind CSS v4 (CSS-first 설정, `@tailwindcss/vite` 플러그인)
- **라우터:** React Router v7 (SPA 모드)
- **유틸리티:** clsx + tailwind-merge (cn 유틸)
- **린트/포매팅:** Biome
- **테스트:** Vitest + Testing Library
- **배포:** Vercel (프론트엔드) + Supabase (백엔드)

## 6. 비기능 요구사항

### 성능 요구사항
- 초기 로딩 시간(LCP): 2.5초 이내 (코드 스플리팅 및 이미지 지연 로딩 적용).
- 렌더링 퍼포먼스: 데이터 그리드 스크롤 시 60FPS 유지.
- 리렌더링 최적화: React.memo, useMemo, useCallback 활용으로 복잡한 필터 조작 시 재계산 비용 최소화.

### 보안 요구사항
- 모든 테이블에 RLS(Row Level Security) 정책 활성화.
- 환경변수로 시크릿 관리, 코드 하드코딩 금지.
- OWASP Top 10 준수.
- 사용자 입력 검증 및 새니타이즈.

### 접근성 요구사항
- 시맨틱 HTML 사용.
- 키보드 탐색 지원.
- 색상 대비 비율 최소 4.5:1 유지.
