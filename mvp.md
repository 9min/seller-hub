# 📄 PRD: 커머스 파트너 통합 관리 대시보드 (가칭: Seller-Hub MVP)

## 1. 프로젝트 개요 (Overview)
* **목적:** 대형 커머스 플랫폼에 입점한 브랜드 파트너(셀러)가 상품, 주문, 매출 데이터를 한눈에 파악하고 효율적으로 비즈니스를 운영할 수 있도록 지원하는 B2B 어드민 웹 애플리케이션 구축.
* **기술적 목표:** * 수만 건의 대규모 데이터를 브라우저 환경에서 지연 없이 처리하는 렌더링 성능 최적화.
  * 복잡한 상태(다중 필터, 정렬, 페이지네이션)를 안정적으로 관리하는 프론트엔드 아키텍처 설계.
  * 기획부터 디자인, 기저 인프라 연동, 배포에 이르는 E2E(End-to-End) 사이클 주도.

## 2. 대상 유저 (Target User)
* **커머스 입점 브랜드 관리자 및 MD:** 하루 수천 건 이상의 주문을 처리하고, 실시간 정산 데이터 및 광고 효율 지표를 확인하여 비즈니스 의사결정을 내려야 하는 실무자.

## 3. 주요 기능 및 요구사항 (Key Features)

### 3.1. 비즈니스 인사이트 대시보드 (Overview Dashboard)
셀러가 로그인 시 가장 먼저 마주하는 화면으로, 비즈니스 핵심 지표를 직관적으로 시각화합니다.
* **주요 지표 위젯:** 일간/주간 총 매출, 신규 주문 건수, 배송 지연 건수, 반품/교환율 실시간 표출.
* **데이터 시각화 (Charts):**
  * **매출 추이 라인 차트:** 기간별(일/주/월) 매출액 추이 변화 시각화.
  * **카테고리별 판매 비중 파이 차트:** 인기 상품 및 카테고리 분포도 제공.
* **요구사항:** 차트 렌더링 시 애니메이션 최적화를 수행하고, 윈도우 리사이징에 유연하게 반응하는 반응형(Responsive) 레이아웃 적용.

### 3.2. 고성능 대용량 데이터 그리드 (Complex Data Grid)
이 프로젝트의 기술적 핵심으로, 무거운 DOM 트리를 효율적으로 핸들링하는 구간입니다.
* **주문/상품 리스트 렌더링:** 최소 5만 건 이상의 더미 데이터를 테이블 형태로 렌더링.
* **가상화 (Virtualization):** 화면에 보이는 영역의 로우(Row)만 렌더링하여 DOM 노드 갯수를 제한하고 메모리 누수 및 스크롤 버벅임 방지.
* **복합 필터링 및 정렬:**
  * 기간 설정 (Date Picker), 주문 상태 다중 선택 (결제완료, 상품준비중, 배송중 등), 텍스트 검색 (상품명, 구매자명).
  * 각 컬럼별 오름차순/내림차순 다중 정렬 기능.
* **상태 동기화:** 사용자가 새로고침하거나 링크를 공유해도 필터 상태가 유지되도록 URL 쿼리 파라미터(Query Parameter)와 전역 상태를 동기화.

## 4. 기술 스택 및 시스템 아키텍처 (Architecture)
* **Frontend:** React, TypeScript. 상태 관리는 Zustand(또는 Redux-Toolkit), 데이터 페칭은 SWR(또는 React Query) 적용.
* **Backend & DB (BaaS):** Supabase (PostgreSQL 기반 데이터베이스 구성 및 실시간 데이터 구독 기능 활용).
* **Deployment:** Vercel을 통한 CI/CD 파이프라인 구축 및 Edge Network 배포.
* **UI/UX:** 확장성을 고려한 커스텀 Design System 구축 및 TailwindCSS 연동.

## 5. 'AI 네이티브' 개발 전략 (Productivity Strategy)
개발 생산성을 극대화하여 비즈니스 로직과 최적화에 집중하는 'AI 네이티브' 방식을 적용합니다.
* **디자인-코드 자동화:** Figma MCP 등을 연동하여 기획된 대시보드 UI 디자인을 코드로 빠르게 변환, 마크업 시간 단축.
* **워크플로우 자동화 (Claude Code):** * Supabase 데이터베이스 스키마 설계 및 대용량 더미 데이터(주문 내역, 상품 정보) 생성 스크립트 작성 자동화.
  * 반복적인 API 연동 보일러플레이트 코드 및 단위 테스트 코드 자동 생성.

## 6. 성능 최적화 목표 지표 (Performance KPIs)
* **초기 로딩 시간(LCP):** 2.5초 이내 달성 (코드 스플리팅 및 이미지 지연 로딩 적용).
* **렌더링 퍼포먼스:** 데이터 그리드 스크롤 시 60FPS 유지 (React Profiler 및 Chrome DevTools Performance 탭으로 검증).
* **리렌더링 최적화:** React.memo, useMemo, useCallback의 적절한 활용으로 복잡한 필터 조작 시 차트 및 테이블의 재계산 비용 최소화.
